# 游戏适配标准规范

## 概述

为确保开源游戏能够顺利集成到GameGameBoy.online平台，特制定此适配标准规范。所有选定的开源游戏都需要按照此标准进行适配和优化。

---

## 1. 技术标准

### 1.1 基础技术要求
- ✅ **HTML5标准**: 使用HTML5语义化标签
- ✅ **响应式设计**: 支持桌面端和移动端
- ✅ **现代浏览器兼容**: 支持Chrome、Firefox、Safari、Edge
- ✅ **无第三方依赖**: 避免依赖外部CDN或服务
- ✅ **本地化资源**: 所有游戏资源本地托管

### 1.2 性能要求
- ✅ **加载时间**: 首屏加载 < 3秒
- ✅ **运行性能**: 60fps流畅运行
- ✅ **内存占用**: 峰值内存 < 100MB
- ✅ **资源大小**: 总资源大小 < 10MB
- ✅ **缓存策略**: 合理使用浏览器缓存

### 1.3 兼容性要求
- ✅ **桌面端**: 1920x1080 到 1024x768
- ✅ **移动端**: 414x896 到 375x667
- ✅ **触控支持**: 完整的触摸操作支持
- ✅ **键盘支持**: 完整的键盘操作支持
- ✅ **全屏支持**: 支持全屏模式

## 2. 文件结构标准

### 2.1 游戏目录结构
```
games/[category]/[game-name]/
├── index.html              # 主游戏文件
├── css/
│   └── style.css          # 游戏样式
├── js/
│   ├── game.js           # 游戏逻辑
│   ├── engine.js         # 游戏引擎(如使用)
│   └── utils.js          # 工具函数
├── assets/
│   ├── images/           # 图片资源
│   ├── sounds/           # 音频资源
│   └── fonts/            # 字体文件
├── README.md             # 游戏说明
└── LICENSE               # 许可证文件
```

### 2.2 文件命名规范
- **HTML文件**: 使用kebab-case命名 (例: `gold-miner.html`)
- **CSS文件**: 使用kebab-case命名 (例: `game-style.css`)
- **JS文件**: 使用kebab-case命名 (例: `game-logic.js`)
- **资源文件**: 使用描述性命名 (例: `player-sprite.png`)

### 2.3 代码规范
- **HTML**: 使用语义化标签，包含适当meta标签
- **CSS**: 使用BEM命名规范，避免全局样式污染
- **JavaScript**: 使用ES6+语法，模块化编程
- **注释**: 关键逻辑添加注释，便于维护

### 4.2 JavaScript集成标准
```javascript
// 游戏初始化
document.addEventListener('DOMContentLoaded', function() {
    // 初始化游戏
    const game = new Game({
        container: 'game-container',
        width: 800,
        height: 600,
        responsive: true
    });

    // 全屏功能
    const fullscreenBtn = document.getElementById('fullscreen-btn');
    fullscreenBtn.addEventListener('click', toggleFullscreen);

    // 游戏控制
    game.init();
    game.start();
});

// 全屏功能
function toggleFullscreen() {
    const gameContainer = document.getElementById('game-container');

    if (!document.fullscreenElement) {
        gameContainer.requestFullscreen();
    } else {
        document.exitFullscreen();
    }
}
```

## 10. 评估标准

### 10.1 评分标准
- **技术实现** (30分): 代码质量、性能、兼容性
- **用户体验** (25分): 界面设计、操作体验、响应速度
- **功能完整性** (25分): 功能完整、无bug、稳定运行
- **集成度** (20分): 与网站集成度、数据一致性

### 10.2 通过标准
- **总分 ≥ 80分**: 可以直接集成
- **总分 60-79分**: 需要修改后集成
- **总分 < 60分**: 不推荐集成
