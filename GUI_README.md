# 🎮 游戏截图管理系统 GUI

这是一个为GameGameBoy游戏网站设计的截图管理系统图形用户界面。

## ✨ 功能特性

### 📸 批量截图功能
- **批量生成截图**: 为所有缺少截图的游戏生成截图
- **强制替换截图**: 重新生成指定分类的所有游戏截图
- **完整截图系统**: 运行完整的截图处理流程

### 🎯 单个游戏管理
- **游戏选择器**: 从下拉菜单选择特定游戏
- **单个截图**: 为选中的游戏生成截图
- **实时状态**: 显示当前处理的进度和状态

### 📊 统计和工具
- **截图统计**: 显示总体和各分类的截图完成情况
- **文件夹管理**: 快速打开截图存储文件夹
- **缓存清理**: 清理Playwright和其他缓存文件

### ⚙️ 可配置参数
- **批次大小**: 设置每次处理的游戏数量 (1-100)
- **超时时间**: 设置单个游戏的截图超时时间 (10-120秒)

## 🚀 使用方法

### 方法一：使用启动脚本（推荐）
```bash
./run_screenshot_gui.sh
```

### 方法二：手动启动
```bash
# 激活虚拟环境
source venv/bin/activate

# 运行GUI程序
python screenshot_gui.py
```

### 必需依赖
- Python 3.7+

## 🖥️ 界面布局

- **系统状态**: 显示截图服务的可用性
- **批量截图**: 批量处理功能按钮

- **进度控制**: 配置处理参数

### 底部状态栏
- **当前状态**: 显示程序运行状态

## 🎮 使用流程

### 1. 批量生成截图
1. 点击 "📸 批量生成截图"
2. 系统自动处理所有缺少截图的游戏
3. 在右侧日志面板查看进度
4. 可随时点击 "⏹️ 停止" 中断处理

### 2. 单个游戏截图
1. 从下拉菜单选择游戏
2. 点击 "📷 截图选中游戏"
3. 等待处理完成

### 3. 强制替换截图
1. 选择一个游戏确定要处理的分类
2. 点击 "🔄 强制替换截图"
3. 系统重新生成该分类所有截图

### 4. 查看统计信息
1. 点击 "📊 统计截图状态"
2. 查看详细的完成情况报告

## 🔧 配置选项

### 批次大小
- **范围**: 1-100
- **推荐值**: 15-20
- **说明**: 每次同时处理的游戏数量，值越大速度越快但占用资源越多

### 超时时间
- **范围**: 10-120秒
- **推荐值**: 30秒
- **说明**: 单个游戏截图的最大等待时间

## ⚠️ 注意事项

1. **虚拟环境**: 确保在正确的虚拟环境中运行
2. **依赖安装**: 所有Python包需在虚拟环境中安装
3. **存储空间**: 确保有足够的磁盘空间存储截图
4. **网络连接**: 截图过程需要访问游戏URL，确保网络连接正常
5. **处理时间**: 批量处理可能需要较长时间，建议分批处理

## 🐛 故障排除

### 常见问题

**Q: 提示"截图服务不可用"**
A: 检查相关Python文件是否存在，确保虚拟环境正确激活

**Q: 程序启动失败**
A: 检查tkinter是否安装，在Linux上运行 `sudo apt-get install python3-tk`

**Q: 截图生成失败**
A: 检查网络连接和URL有效性，查看日志获取具体错误信息

**Q: 程序卡死**
A: 增加超时时间，或减小批次大小

### 获取帮助
- 查看右侧日志面板的错误信息
- 检查虚拟环境和依赖安装
- 确认文件权限和目录结构
